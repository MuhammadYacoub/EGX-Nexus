# `dashboard-api` Microservice

## Role & Responsibility
**REST API for UI and Dashboard**
The `dashboard-api` is responsible for querying data from the databases (MySQL, TimescaleDB, Redis) and formatting it to be easily consumed by the eventual frontend UI (Phase 8).

## Technical Specifications
- **Stack:** Node.js
- **Internal Port:** 3007
- **Status:** Scaffolded. Phase 8 Target.

## Projected Capabilities (Phase 8)
- Expose endpoints for real-time signal displays.
- Visualizations for agent debate reasoning (bull vs. bear arguments).
- Portfolio tracking capabilities.
- Historical decision accuracy analytics.

## Notes for AI Agents
- **Future Readiness:** All API endpoints must adhere to REST principles and provide clean, structured JSON to ensure smooth frontend integrations later.
